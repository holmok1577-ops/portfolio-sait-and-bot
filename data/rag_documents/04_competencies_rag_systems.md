# Компетенции: RAG-системы

## Что такое RAG

RAG, или Retrieval-Augmented Generation, - это подход, при котором языковая модель сначала получает релевантные фрагменты из базы знаний, а затем отвечает на вопрос пользователя с учетом найденного контекста.

## Как работает RAG

1. Документы готовятся и приводятся к понятной структуре.
2. Текст разбивается на фрагменты.
3. Фрагменты превращаются в embeddings.
4. Embeddings сохраняются в векторной базе.
5. При вопросе пользователя система ищет релевантные фрагменты.
6. Модель формирует ответ на основе найденного контекста.

## Стек в текущем проекте

- Backend: Python, FastAPI.
- Векторное хранилище: ChromaDB.
- Модель для ответов: OpenAI API через ProxyAPI.
- Embeddings: text-embedding-3-small.
- Интерфейс: чат-виджет на сайте и Telegram-бот.

## Преимущества RAG

- Ответы основаны на документах.
- Базу знаний можно обновлять без переобучения модели.
- Можно указывать источники.
- Подходит для FAQ, инструкций, регламентов, портфолио и внутренних знаний.
